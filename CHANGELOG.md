**Refactor: dedup & primitives (Phases 3, 4)**

- New frontend primitives in `frontend/src/components/primitives/`: `Unit`, `MetricRow`, `DetailCardHeader`, `PlateNavigationChip`, `FormattedVolume`. Killed 11× metric-row duplication in the results page.
- Group color palette consolidated to `frontend/src/styles/tokens.ts` (was duplicated in `PlateVisualization` and `exportUtils`).

**Refactor: types & cleanup (Phases 5, 6)**
- Typed API responses with `SeedingRow`, `DyeRow`, `IMetaRow`, `FormattedSummaryRow` interfaces; eliminated the last `any` cast in `DesignPage`.
- Modernized backend typing: `Dict`/`List`/`Optional` from `typing` replaced with `dict`/`list`/`| None`. Added missing return types on `clean_values`, `health_check`, `run_calculation`, `upload_csv_files`, `create_temp_config`.

**Refactor: file splits & directory organization (Phases 7, 11)**
- `frontend/src/utils/exportUtils.ts` (614 LOC) split into 5 focused modules under `utils/export/`: `svgExport`, `pngExport`, `csvLayoutExport`, `colorPalette`, `plateLayout`, `downloadFile`, `downloadFilenames`. The existing `exportUtils.ts` now is a 30-line barrel re-export.
- `frontend/src/pages/DesignPage.tsx` (1,432 LOC) split into 17 files under `pages/design/`: orchestrator + 4 panels (`ConfigBar`, `ParametersPanel`, `CellsModePanel`, `DyesModePanel`) + 6 modals (`Group`, `EditGroup`, `ConfirmProcess`, `DyeWarning`, `DyeAssignment`, `ManageDyeProgram`) + shared `AddDyeForm` sub-component + `DownloadActions` + 2 hooks (`useDesignKeyboard`, `useDownloadHandlers`) + types.
- `frontend/src/components/PlateVisualization.tsx` (597 LOC) split: `PlateLegend.tsx` extracted.
- `frontend/src/components/ProtocolSection.tsx` (702 LOC) split: `ProtocolDetailCard.tsx` and shared `protocolTypes.ts` extracted.
- `frontend/src/components/ResultsDisplay.tsx` (589 LOC) had `getWellRange` and the hover-detail builders extracted to `utils/wellRange.ts` and `utils/wellHoverDetails.ts`.
- Phase 11 directory reorganization: components grouped by domain (`components/{plate,protocol,inputs,primitives}/`), pages co-located with their companions (`pages/design/`, `pages/results/`), 8-file export pipeline grouped under `utils/export/`.
- After both phases, no source file >500 LOC. ESLint `max-lines` warning list is empty.

### Changed
- Pinned Docker base images to patch level: `python:3.11.10-slim-bookworm` and `node:20.18.0-slim`.
- `pyproject.toml` declares `pandas>=2.0,<3` (loose range for library consumers); `backend/requirements.txt` continues to pin the install version exactly.
- Moved `jupyterlab` + `ipykernel` install behind a Dockerfile build arg `INSTALL_NOTEBOOK_DEPS` (default `true`, preserves current behavior).
- `scripts/start.sh` no longer rebuilds on every invocation. First run auto-builds; subsequent runs reuse the cached image. Pass `--build` to force a rebuild after code changes.
- New `scripts/stop.sh` replaces the old `scripts/start.sh stop` subcommand.
- Frontend documentation deduplicated: deleted `frontend/DEVELOPER.md` and `frontend/ORGANIZATION.md` (content was covered by `CONTRIBUTING.md` and `docs/repo-structure.md`); slimmed `frontend/README.md` to a small frontend-specific config pointer.

### Fixed
- `µL` rendering on the Design page parameter labels (`Final Well Volume`, `Dead Volume — Cell Suspension`, `Dead Volume — Dye`). Root cause: `text-transform: uppercase` was case-mapping U+00B5 (MICRO SIGN) to U+039C (GREEK CAPITAL LETTER MU), which renders as a Latin "M". Introduced a small `<Unit>` primitive with `text-transform: none`.
- `pandas` version mismatch between `requirements.txt` (`==2.0`) and `pyproject.toml` (`>=2.0`).
- `eslint` was missing from `frontend/package.json` `devDependencies` despite being referenced by the lint script.
- Dead `// eslint-disable-line` directive in `DesignPage.tsx`.
