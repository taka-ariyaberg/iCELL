## Python — production (`backend/requirements.txt`)

These are the seven direct production dependencies installed into the FastAPI app container. Top-level pins live in `backend/requirements.txt`; full transitive resolution is captured in `backend/requirements.lock` (generated by `pip-compile`).

| Package | Version | Purpose |
|---|---|---|
| `fastapi` | `0.104.1` | Async web framework — implements the REST API (`/api/health`, `/api/run`, `/api/upload-csv`) |
| `uvicorn` | `0.24.0` | ASGI server that hosts the FastAPI application |
| `python-multipart` | `0.0.6` | Multipart form parser — required by FastAPI for the CSV upload endpoint |
| `numpy` | `1.26.4` | Numerical arrays — pinned because pandas pulls it transitively |
| `pandas` | `==2.0` (requirements.txt) / `>=2.0,<3` (pyproject.toml) | Tabular data and CSV I/O — used pervasively in `src/icell/processing/` and `src/icell/reporting/` |
| `pydantic` | `2.5.0` | Request/response validation — defines the schemas in `backend/api/schemas.py` |
| `pydantic-settings` | `2.1.0` | Environment-driven configuration loader — companion to pydantic |

**Reproducibility:** `backend/requirements.lock` pins every transitive dependency. Regenerate with:

```bash
docker run --rm -v "$PWD/backend:/app" -w /app python:3.11.10-slim-bookworm sh -c \
  "pip install --quiet pip-tools && pip-compile --quiet --output-file=requirements.lock requirements.txt"
```

## How to add a new dependency

1. Add the package + version to its source-of-truth file (`requirements.txt`, `pyproject.toml`, or `frontend/package.json`).
2. Regenerate the relevant lockfile (`requirements.lock` or `package-lock.json`) using the command in the relevant section above.
3. Add a row to this document — name, version, purpose, source-of-truth file.
4. Run the smoke test (`docker compose build` plus `npm run lint && npm test` for frontend deps).

If a dependency is added without updating this file, treat it as a documentation regression.
